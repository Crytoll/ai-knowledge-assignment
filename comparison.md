# 裸 API vs OpenCode 对比体会
## 实验过程
- 裸 API 调用：回复显示“我来分析项目的目录结构和代码质量。不过由于您没有提供具体的项目代码和目录结构，我将给出一个通用的分析框架和改进建议。”，答非所问，说了一堆正确的废话。
- OpenCode 编排：给出了项目总览、目录结构评价、代码质量问题、改进建议，并能根据使用者的选择真正执行落地
## 关键差异
- [差异 1：是否能读文件]裸 API 调用是在云上，读不了本地项目文件；OpenCode 已装在本机，可以读文件。
- [差异 2：是否有上下文感知]裸 API 调用无上下文感知，一轮调用结束后上下文也就结束了；OpenCode 有上下文感知，右上方能看到 Context 使用情况。
- [差异 3：建议是否具体]裸 API 调用给的是通用框架，没有具体方案；OpenCode 给的是可进一步执行的具体 Action。
## 我对「无状态推理」和「有状态编排」的理解
裸 API 调用是无状态推理，不能读写文件，没有上下文感知，没有具体建议；OpenCode 是有状态编排，能读写文件，有上下文感知，能给具体建议并真实执行。
